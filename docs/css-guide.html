Поле «Свой CSS», объяснённое
на пальцах

Вкладка «Твики» в YMHub даёт текстовое поле, в которое можно вписать любое правило CSS и изменить интерфейс Яндекс Музыки так, как не умеет ни один из 9 готовых переключателей. Этот гайд — для тех, кто ни разу не писал CSS и хочет разобраться по шагам.

1Из чего состоит одна строчка CSS

Любое правило CSS выглядит одинаково: «кому» — «что» — «на сколько». Вот реальный пример, который можно вставить в «Свой CSS» прямо сейчас — он прячет цветной анимированный фон на странице «Моя волна»:

[data-test-id="VIBE_ANIMATION"]селектор — кому { displayсвойство — что: noneзначение — на что !important; }

Селектор — какому элементу на странице адресовано правило. Свойство — что именно меняем (видимость, размер, цвет...). Значение — на что меняем. !important означает «применить обязательно, даже если у Яндекс Музыки уже есть своё правило на этот счёт» — без него ваше правило иногда может быть тише собственных стилей страницы и просто не подействовать.

⚠

Важно: если забыть точку с запятой или одну из скобок { } — браузer не покажет никакой ошибки. Правило просто молча не применится. Если что-то «не работает» — первым делом проверьте именно это.

2Как найти свой селектор

Селектор для готовых примеров ниже уже есть, но рано или поздно вы захотите спрятать или изменить что-то своё. YMHub уже открывает технический доступ к странице Яндекс Музыки (через него же работают лайки и твики), и этим можно воспользоваться, чтобы посмотреть на разметку страницы вживую — без установки чего-либо ещё.

1
Запустите YMHub и Яндекс МузыкуПодождите несколько секунд, пока YMHub подключится (плеер на вкладке «Плеер» должен показать текущий трек).
2
Откройте обычный Chrome и перейдите на 127.0.0.1:9876Это локальный адрес внутри вашего компьютера — откроется список страниц, к которым подключён YMHub.
3
Кликните по странице «Яндекс Музыка»Откроются настоящие инструменты разработчика Chrome — те же, что встроены в любой браузер.
4
Включите инструмент «Inspect»Иконка курсора в левом верхнем углу панели DevTools (или Ctrl+Shift+C), затем кликните по нужному элементу прямо на странице Яндекс Музыки.
5
Посмотрите на подсветившийся HTMLТам будет что-то вроде class="qQ7GQU14Ekgg AlbumCover_cover__DhS1R" — нужная часть для селектора находится именно тут.

3Главная ловушка — «случайные» классы

Если в инспекторе вы увидели класс вроде AlbumCover_cover__DhS1R — последняя часть после двойного подчёркивания (DhS1R) не опечатка и не что-то особенное лично для вас. Это случайный набор символов, который Яндекс генерирует при каждой сборке приложения — и он меняется при следующем же обновлении Яндекс Музыки.

// Сломается при следующем обновлении ЯМ — хэш «DhS1R» поменяется: .AlbumCover_cover__DhS1R { display: none !important; } // Переживёт обновление — мы просим "содержит", а не "равно ровно этому": [class*="AlbumCover_cover__"] { display: none !important; }

Квадратные скобки [class*="..."] означают «класс, который содержит эту подстроку» — а не «равен ей в точности». Достаточно взять стабильную часть имени до __ и отбросить случайный хвост — тогда правило продолжит работать и после обновлений, пока Яндекс не переименует сам компонент целиком (а это случается заметно реже, чем пересборка).

💡

Бонус-приём: иногда у элемента в инспекторе есть атрибут data-test-id или data-testid — такие атрибуты почти всегда стабильнее любого класса (они существуют специально для автотестов Яндекса и переименовываются гораздо реже). Если видите такой атрибут — берите его вместо класса.

4Готовые примеры — просто скопируйте

Шесть рабочих правил, которые можно вставить в поле «Свой CSS» прямо сейчас. Можно вписать сразу несколько — каждое на новой строке, как в обычном CSS-файле.

Скрыть AI-комментарий под плеером

класс + хэш
[class*="VibePage_words__"]{display:none!important;}
Та самая подсказка со искрой под названием трека на «Моей волне».

Скрыть цветной анимированный фон

data-test-id
[data-test-id="VIBE_ANIMATION"]{display:none!important;}
Атрибут вместо класса — переживёт обновления Яндекс Музыки без правок.

Скрыть плашку «Вышла новая версия»

класс + хэш
[class*="MainPage_betaSlot__"]{display:none!important;}
Кнопка с заметками об обновлении в углу главного экрана.

Скрыть Плюс-бейдж в профиле

data-test-id
[data-test-id="USER_PROFILE_PLUS_BADGE"]{display:none!important;}
Ссылку на подписку рядом с именем профиля в сайдбаре.

Скрыть конкретный раздел меню

data-test-id
[data-test-id="NAVBAR_NAVIGATION_ITEM_CONCERTS"]{display:none!important;}
Здесь это «Концерты» — для «Книг и подкастов» возьмите NAVBAR_NAVIGATION_ITEM_NON_MUSIC, для «Для вас и Тренды» — NAVBAR_NAVIGATION_ITEM_FOR_YOU_AND_TRENDS.

Сделать обложку трека круглой

класс + хэш
[class*="AlbumCover_cover__"]{border-radius:50%!important;}
CSS — это не только «скрыть». Тот же селектор, что у твика «Крупная обложка», но другое свойство.

5Частые вопросы

Я вставил правило, но ничего не изменилось

Сначала проверьте точку с запятой и обе скобки { } — опечатка в CSS никогда не показывает ошибку, просто тихо ничего не делает. Если синтаксис в порядке — возможно, классу нужен !important, либо после обновления Яндекс Музыки изменился хэш в классе: откройте инспектор (раздел 2) и проверьте селектор заново.

Как отменить изменения и вернуть всё как было

Просто очистите поле «Свой CSS» и кликните в любое другое место — правило применяется заново при каждом изменении, а пустое поле ничего не добавляет к встроенным твикам.

Можно ли что-то сломать насовсем?

Нет. Это обычный CSS, который существует только в окне приложения, пока работает YMHub — он ничего не отправляет на серверы Яндекса и не трогает файлы самой программы. Если результат не понравился — очистите поле, и всё вернётся в исходный вид.

А JavaScript или анимации можно?

Нет, поле принимает только правила CSS, без какой-либо логики или скриптов — это сделано специально, чтобы поле было безопасным для произвольного текста.

Почему именно 127.0.0.1:9876?

YMHub подключается к Яндекс Музыке через стандартный Chrome DevTools Protocol — на этом порту он открывает обычный, ничем не урезанный интерфейс инструментов разработчика. Это тот же порт, которым пользуется сама DLL внутри YMHub, просто открытый и для вас.
